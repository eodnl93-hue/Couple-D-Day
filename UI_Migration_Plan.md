# 커플 디데이 앱 UI 마이그레이션 기획안

기존 `reference` 폴더에 구현된 React 기반의 UI를 분석하고, 이를 현재 Flutter 프로젝트에 반영하기 위한 기획서입니다. **"UI의 절대 변형이 없어야 한다"**는 요구사항에 맞춰 두 가지 방안을 제안합니다.

---

## 1. 기존 UI 분석 (React + Tailwind CSS + Framer Motion)

현재 만들어진 화면은 총 4개이며, 다음과 같은 특징을 가집니다.

*   **StartScreen**: 앱의 첫 화면. 프로필 이미지, 닉네임, 시작일을 보여주며 부드러운 애니메이션 효과가 포함되어 있습니다.
*   **MainScreen**: D-Day 카운터와 함께 커플의 추억(사진 + 메모)을 스크롤 타임라인 형태로 보여주는 피드(Feed) 화면입니다.
*   **SettingsScreen**: 프로필 이미지, 닉네임, 시작일을 수정하고 저장할 수 있는 세팅 화면입니다.
*   **PostCreationScreen**: 새로운 추억 사진과 메모를 등록/수정하는 화면입니다.

**데이터 구조 (Types)**
*   `CoupleProfile`: 닉네임(nickname), 시작일(startDate), 프로필이미지(profileImage)
*   `Post`: 추억 기록. 사진, 날짜, 메모, 좋아요(isLiked) 등의 정보를 포함

**핵심 디자인 요소**
*   테마 색상: Pink (#ec4899), Gray 계열 바탕색 (Tailwind 기본 색상)
*   아이콘: Lucide 스타일 아이콘 모음 (하트, 설정, 홈 등)
*   애니메이션: 화면 전환 시 스케일(Scale) 및 투명도(Opacity) 효과, 하단 메뉴 버튼 플로팅 쉐도우 효과

---

## 2. Flutter 적용 방안 제안

요구하시는 "UI의 형태가 100% 완벽히 보존" 되어야 한다는 점을 고려해 두 가지 방안을 기획했습니다. 둘 중 어떤 방식을 선호하시는지 선택해 주시면 코드 작성을 진행하겠습니다.

### 옵션 A: WebView 연동 (픽셀 퍼펙트 - 100% 완벽 일치 보장)
현재 잘 만들어진 웹앱(React) 프로젝트를 빌드하여, 그 결과물(HTML/CSS/JS)을 Flutter 앱 내에 오프라인 상태로 포함시킵니다. `webview_flutter` 기능이나 `인앱 브라우저` 패키지를 통해 웹앱 화면을 띄워주는 방식입니다.

*   **장점**: 사용자가 만든 Web UI(Tailwind, Framer Motion)가 **단 1픽셀의 오차도 없이 똑같이** 출력됩니다.
*   **단점**: 네이티브 모바일 앱이라기보다는 브라우저 위에서 동작하는 형태이므로, 모바일 시스템 제스처 대응(스와이프 등) 등에서 네이티브보다는 제약이 있을 수 있습니다.

### 옵션 B: 완전한 Flutter 코드로 변환 (네이티브 성능과 자연스러움)
현재 웹앱 UI의 디자인 코드(Tailwind 속성값, 마진, 색상, 애니메이션 등)를 1:1로 분석하여 **100% 동일한 모습의 Flutter 위젯으로 모두 새로 제작**하는 방식입니다.

*   **장점**: 앱 구동이 훨씬 가볍고 빠릅니다. 향후 모바일 하드웨어(카메라 앨범 접근 등) 연동이나 아키텍처 확장이 용이한 '진짜 Flutter 구조'가 됩니다.
*   **어떻게 100% 구현할 것인가?**
    *   디자인: `Container`의 BoxShadow, Radius, Color를 기존 Tailwind의 Hex 값과 완벽히 동일하게 작성합니다.
    *   아이콘: `lucide_icons` 패키지를 사용하여 웹에서 사용한 동일한 아이콘을 붙여넣습니다.
    *   애니메이션: Flutter의 여러 애니메이션 관련 위젯(`AnimatedSwitcher` 등 혹은 `flutter_animate` 패키지)을 이용하여 Framer Motion의 효과(Scale/Opacity)를 완벽히 모방합니다.
*   **단점**: 개발 엔진(Web vs Native)에 따른 글꼴 렌더링 방식의 미세한 차이에 의해 자간/행간 등에서 약 1% 이하의 아주 미묘한 시각적 차이가 있을 수 있지만, 기본적으로 동일하게 구현할 수 있습니다.
